Version: 1
- Каркас FastAPI (app.py) с GET/HEAD для / и /health (для UptimeRobot).
- На старте пинг Binance через /api/v3/account (HMAC) для проверки ключа.
- Отправка в ADMIN_CHAT_ID: "<UTC> Бот запущен\nBinance connection: ✅/❌".
- Используются переменные из Environment Variables.txt (прокси, токен, admin chat, ключи). Новых переменных не добавлял.
- Совместимо с:
  Build Command=pip install -r requirements.txt
  Start Command=uvicorn app:app --host 0.0.0.0 --port $PORT

Version: 2
- Добавлен Telegram webhook: на старте сверяем getWebhookInfo и ставим setWebhook на WEBHOOK_BASE + '/tg'.
- Обработчик /tg принимает POST от Telegram и отвечает на /start и любые тексты.
- Уведомление админу теперь включает статус вебхука (Webhook: ✅/❌).
- Прокси берутся из HTTP_PROXY/HTTPS_PROXY из окружения (как и раньше).
- Остальное без изменений: HEAD для / и /health, Binance ping на старте.

Version: 3
- Команда /portfolio: первая реализация — спот+earn из Binance, цены через тикеры *USDT.
- Формат ответа был в Markdown (```code```), сортировка по USD.

Version: 4
- Исправлен импорт json в app.py, добавлены таймауты httpx, мелкие правки логирования.
- В /portfolio добавлены попытки резервного запроса цен, фильтрация балансов < 1$.

Version: 5
- Устранены ложные падения при разборе Markdown у Telegram (fallback в plain).
- Мелкие правки обработки прокси и HEAD-ответов.

Version: 6
- Нормализация активов: LDUSDC→USDC, WBETH/BETH→ETH, WBTC/BTCB→BTC (и др.).
- Стейблы расширены: USDT/USDC/BUSD/FDUSD/USDP/DAI/TUSD/AEUR/EUR и др. — показываются внизу.
- % теперь считается от суммы всех НЕстейбл-активов (BTC+ETH+BNB+…).
- Порядок: BTC, ETH сверху, далее альткоины по USD, затем стейблы, потом Earn.
- Формат вывода строго как в примере и обёрнут в HTML <pre> (в app.py parse_mode=HTML) — Telegram больше не ругается на Markdown.
- Код portfolio.py переписан без внешних зависимостей, только httpx.
